## Image Guidelines

### Format Recommendations
- **WebP format** for photos (best compression and quality)
- **PNG format** for logos and icons with transparency
- **SVG format** for simple graphics and icons (scalable)

### Size Optimization
- Compress all images before uploading
- Use responsive images with multiple sizes
- Implement lazy loading for below-the-fold images

### Naming Convention
- Use lowercase letters
- Use underscores instead of spaces
- Use descriptive names
- Include size suffix if multiple versions (e.g., `logo_small.png`, `logo_large.png`)

### Recommended Tools
- **TinyPNG** - https://tinypng.com/ (PNG/JPG compression)
- **Squoosh** - https://squoosh.app/ (WebP conversion)
- **ImageOptim** - https://imageoptim.com/ (Mac app for optimization)
- **GIMP** - https://www.gimp.org/ (Free image editor)

## Image Specifications

### Logo Header
- Format: PNG with transparency
- Size: 200x80px (2x for retina: 400x160px)
- Max file size: 50KB

### Hero Images
- Format: WebP (with JPG fallback)
- Size: 1920x1080px (Full HD)
- Max file size: 200KB
- Quality: 80-85%

### Service Images
- Format: WebP (with JPG fallback)
- Size: 800x600px
- Max file size: 150KB
- Quality: 80%

### Project Gallery
- Format: WebP (with JPG fallback)
- Thumbnail: 400x300px (max 50KB)
- Full size: 1200x900px (max 200KB)
- Quality: 80-85%

### Icons
- Format: PNG with transparency or SVG
- Size: 64x64px (icons), 512x512px (PWA icons)
- Max file size: 20KB

### Open Graph Image
- Format: JPG
- Size: 1200x630px (Facebook/LinkedIn recommended)
- Max file size: 300KB
- Quality: 85%

## Accessibility

All images should include:
- Descriptive `alt` text in HTML
- Proper contrast for text overlays
- Meaningful file names
